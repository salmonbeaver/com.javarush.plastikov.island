1 Запрограммировать модель острова 
2 Настройи - "с изменяемыми параметрами"
	2.1 Настройка количесва локаций(клеток) - например, 100х20 
3 Остров состоит из локаций(клеток) "состоящую из массива локаций" - массив двухмерный?
4 Локации будут заполнены растительностью и животными
5 СТАРТ ПРОГРАММЫ - должно быть создано минимум 10 видов травоядных и 5 видов хищников


✅не абстрактный Plant (Растение)
✅abstract Animal (Животное)

	✅	double weight (Вес)
	✅	int maxSpeed (Макс кол-во клеток за раз)
	✅	double final maxSatiety (Макс насыщение)
	✅	double actualSatiety (Фактическая сытость) - в течение ЖЦ животного, значение этого поля должно уменьшаться (или увеличиваться когда поел)

		метод worker - нечто, что уменьшает значение поля текущей сытости

	⭕	eat(Plant or Animal) { - есть растения и/или других животных
		(если в их локации есть подходящая еда)
		Должен оттолкнуться от вероятности съедания
		Должен проверить свою текущую сытость
		} 
	⭕	move(Location) - передвигаться (в соседние локации) {
		а есть ли место для этого вида в новой локации?
		}
	⭕	chooseDirection(Выбрать направление) {
		должен определить в какую локацию МОЖНО идти
		}
	⭕	reproduce(?) - размножаться (при наличии пары в их локации),
	⭕	die(?) - умирать от голода или быть съеденными.
✅	Herbivores (Травоядные)
✅	Predators (Хищники)

⭕entity.Island (Остров)
⭕Location (Локация) / entity.Cell (Клетка)

✅? data.Data
xml / json / yaml - как их читать в Java
Может быть вероятности съедания можно хранить в настройках

Многопоточность
1 секунда - это 1 день на острове
Один scheduled пул для запуска по расписанию: 
- задания роста растений (вырасти на кол-во)
- задания жизненного цикла животных (worker, eat, reproduce, move)
- и задания вывода статистики по системе (sout) - текущее кол-во животных по видам в целом на острове + вес травы
